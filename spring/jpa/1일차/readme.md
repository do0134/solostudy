## 후기

RDBMS와 ORM에 대해서 복습하고, 이를 왜 병행하는 지를 학습했다.

RDBMS는 객체지향에 비해 코드의 실용성이 떨어지는 편이고, ORM은 편하지만 Entity에 접근할 때 신뢰할 수 없으며, 모든 객체를 미리 로딩할 수 없다는 문제가 존재했다. 
+ 객체 답게 매핑한다면 매핑 작업 횟수가 늘어난다. 

이걸 자바에서는 JPA(Java Persistence Api)로 해결한 것


### 작동방식

Java application과 DB사이에서 쿼리를 대체해준다 -> 패러다임의 불일치를 해소시킴


### 장점

- 객체 중심 개발로 코드 생산성이 높아짐
- 필드 변경시 모든 SQL 수정해야 했던 것과 달리 필드만 추가하면 수정가능
- JPA가 상속을 알아서 SQL문 짜서 처리함
- 연관관계도 저장 + 자유로운 객체 그래프 탐색 가능
- 1차 캐시와 동일성 보장 -> 약간의 조회 성능 향상(같은 트랜잭션 안에서 같은 엔티티 반환)
- 트랜잭션을 지원하는 쓰기 지연 (한 번에 요청 전송)
